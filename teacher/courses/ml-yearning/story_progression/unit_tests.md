# 综合测验节点

> 在 Ch.7 和 Ch.14 之间可按需插入综合测验。测验节点不捆绑到某一课——教师在复习阶段触发。

## 测验 1：偏差-方差诊断（Ch.7 后可用）

**场景**：给定三个模型的训练误差和验证误差数据，让学生诊断每个模型的问题类型（高偏差 / 高方差 / 数据不匹配）。

**教学融入**：
- 三月主导此测验：用"看图说话"的方式——让学生先读学习曲线，再给出诊断。
- 丹恒的补充：如果学生诊断正确但解释不精确，追问诊断依据。
- 叙事标记：测验在圆厅进行，白板上画了三组曲线。

## 测验 2：误差归因全流程（Ch.13 后可选）

**场景**：给定一个 ML pipeline 的完整描述（数据收集 → 特征工程 → 模型选择 → 评估），让学生识别每个环节可能的误差来源，排序优先级。

**教学融入**：
- 丹恒主导此测验：让学生用他教的"误差归因"框架一步步排查。
- 三月的补充：用"看病"类比——"你是医生，这个 pipeline 是你的病人，你的诊断顺序是什么？"

## 注意

- 测验不是"考试"——是苏格拉底教学的延伸。不给分数，追问思路。
- 测验节点不要求必然触发——如果学生在日常教学中已经通过追问展示了足够的理解，可跳过。
