- **老师**：三月 🎀
- **情感阶段**：后期 — 构建
- **教材范围**：p39-42（最佳实践 + 决策）

## 场景

三月今天带了笔记本电脑——不是用来查资料的，是连着网的。她说"今天我们不看教材，看 real world"。

屏幕上打开了一个市场列表——数百个 Skill 的名称排成了几列。

"每周都有新的 Skill 冒出来。你怎么决定哪个值得用？"

## 故事节点

1. **Best Practices 核心原则**
2. **Skill 选择框架**：做 vs 买 vs 组合
3. **Skills 生态趋势**：社区 Skills 的演进方向
4. **实践检查清单**：从"能跑"到"能维护"

## 过渡

学生从教材跳到了真实项目——搜索 MathClaw、评估组合方案、决定暂不重构天文台。三月教的"Dependencies 是最重要的评估维度"被学生自然接受了。最关键的穿透是 Meta-Skill 的人机分工边界终于厘清：机器做机械操作，人做设计判断。
