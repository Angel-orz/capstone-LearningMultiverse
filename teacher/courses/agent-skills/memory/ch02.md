## 对话起点

丹恒用投影上课——屏幕上只有一行字："你为什么相信这个 Skill 是好的？"停顿五秒。然后展开 Evaluation Toolkit 的四道门。

## 关键推理链条

```
第一道门：Trigger → 触发条件对不对？
  ↓
精确匹配 vs 意图匹配（学生选精确匹配）
  ↓
第二道门：Output Quality → 结果对不对？
  ↓
第三道门：Tool Trajectory → 方法对不对？
  ↓
第四道门：Token Budget → 成本高不高？
  ↓
System vs Skill Evaluation Illusion → 你在改进系统还是 Skill？
  ↓
Token Budget Isolation 陷阱 → 隔离测试不可靠
```

## 核心概念

| 概念 | 掌握程度 | 备注 |
|------|---------|------|
| Evaluation Toolkit 四道门 | 概念 ✓ / 应用 ✓ | Trigger→质量→轨迹→预算 |
| 精确匹配 vs 意图匹配 | 概念 ✓ / 应用 ✓ | 学生选精确匹配 |
| System vs Skill Evaluation Illusion | 概念 ✓ / 应用 ~ | 区分改进对象 |
| Token Budget Isolation 陷阱 | 概念 ✓ / 应用 ~ | 共享上下文导致不准确 |

## 帕姆表现

用"按错车厢按钮→不需要分析广播质量"类比 Trigger 作为 eval 的第一道门。

## 课后总结

> **"评估要关注结果是否正确、得到结果的方法是否正确、成本是否高。出问题时先检查 Trigger。"**
