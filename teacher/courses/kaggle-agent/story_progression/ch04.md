# Ch.4 — 地下书库的第二张蓝图

- **老师**：丹恒 🗡️
- **情感阶段**：中期 — 连接
- **暗线密度**：1（碎片 1 触发窗口）
- **教材范围**：Day 2 p26-49（A2A 协议 + 智能体架构模式）

## 场景

白板上还留着上节课三月的"高速公路"图——MCP 是那条连接服务器和服务器的路，A2A 是路上跑的两辆车。她用不同颜色的笔画了箭头，有些地方沾了茶渍。

但你到的时候，地下书库的灯已经亮了。键盘声从楼梯下传来——停了。然后是杯底触桌的轻响。

丹恒已经把那张"高速公路"图抄了一份到自己的黑板上——不是复制，是重画。他加了一层：地面之上是 MCP，地面之下是 A2A 的协议栈。你之前没看到的那层结构。

## 故事节点

1. **过渡**：从三月的高速公路到丹恒的地下工程图——同一张图，垂直切面
2. **A2A 协议深度剖析**：Agent Card、Discovery、Task 生命周期——从"两辆车在路上跑"到"两辆车怎么握手、怎么交换货物、怎么确认送达"
3. **架构演进**：从 Simple Agent → Multi-Agent → Agent Team 的演进路径
4. **Bounded vs Unbounded Domains**：给 Agent 划定边界——什么时候边界是朋友，什么时候是敌人
5. **GOTO Problem**：架构中的"失控转移"模式——以及丹恒的旧笔记里对它的标注

## 暗线窗口

- **碎片 1 触发机会**：在讨论 A2A 协议或 agent 间通信的边界问题时，丹恒可能在纠正某处表述时停一下，说"书上这里不对——或者说，不够完整"。（见 danheng.md 碎片 1）

## 真实课堂记录

**2026-06-22 | 丹恒 🗡️ | 精细模式**

学生从并行/串联架构选择切入，自主判断了"无依赖用并行、有依赖用串联"，并用代码审查系统设计了并行 Multi-Agent + Fail-fast 终止的完整方案。关键抽象：学生自然触及了 Orchestrator 决策边界概念、Bounded vs Unbounded Domain 边界判定。暗线碎片 1 未触发（学生直接选择了正确的架构判断）。

## 过渡

白板上的垂直切面图被丹恒留到了第二天——他说"三月上 Ch.5 可能用得上"。地下书库的灯关了，但黑板上那条 MCP 地面之上、A2A 地面之下的分界线还隐约可见。
