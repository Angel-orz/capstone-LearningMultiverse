# Ch.6 — 地下书库的最后一课

- **老师**：丹恒 🗡️
- **情感阶段**：后期 — 构建（收官）
- **暗线密度**：2（碎片 1 + 碎片 2 双触发窗口）
- **教材范围**：Day 3 p36-62 + Day 4 全篇（安全架构 + 评估 + 生产实践）

## 场景

你从暖房下来的时候，地下书库的灯已经亮了。

今天白板上没有课前的问题。没有新的图。所有过去五课的白板内容——Ch.1 的 Vibe Coding 光谱、Ch.2 的 Harness 组件图、Ch.3 的 MCP 高速公路、Ch.4 的 A2A 垂直切面、Ch.5 的 Skill 积木塔——全部被整整齐齐地缩印在一页 A3 纸上，贴在黑板旁边。

丹恒坐在桌前，保温杯冒着热气。桌面上没有教材——有一叠笔记，封面上写着你的名字。

他看了一眼你进来的方向。

"最后一课。不讲新概念——讲保护。"

## 故事节点

1. **五课总览**：一张 A3 纸概括整个课程的知识结构——"你已经有了完整的 Agent 工程化图景。最后一课是给这个图景加上护栏。"
2. **7-Pillar Security Architecture**：Agent 系统的七个安全支柱——不只是"防攻击"，是"设计一个不会轻易出错的系统"
3. **沙箱与供应链安全**：代码执行环境、依赖管理、包安全
4. **身份与权限**：Agent 要有自己的身份——最小权限原则
5. **Agent Evaluation**：Output eval + Trajectory eval——你怎么知道你的 Agent 系统在正常工作
6. **Token Budget 与模型路由**：经济学的最后一环

## 暗线窗口

- **碎片 1 （触发窗口）**：讨论"真实世界的数据不会像教材说的那么干净"时，丹恒可能在纠正某处表述时停一下，说"书上这里不对——或者说，不够完整"。语气更沉，更个人。
- **碎片 2 （触发窗口）**：讨论"模型上线后的反馈循环"时，他的解释比教材深入一个层级。说到"反馈循环是——你可能永远不知道"时，他会停顿。手指可能碰触手腕。

## 真实课堂记录

**2026-06-22 | 丹恒 🗡️ | 精细模式 | 最终课**

课程从全课程 A3 总览图开始。学生自主提出了数据保护和系统安全两个核心关注点。关键穿透：学生理解了 High-Risk Authorization 模式——"人在回路中"——敏感操作前 Agent 向用户提交执行计划。课程以丹恒移交六课手写教学笔记结束。暗线碎片 1 和碎片 2 均未触发（课程偏安全设计原则方向）。

## 过渡

课程结束后，丹恒把那叠写着你名字的笔记递给你。不是教材——是他这六课期间整理的教学笔记，手写的。

"用得上。用不上也没关系——至少留个记录。"

暖房的方向传来三月的声音——她在收拾积木盒。

"丹恒！你们结束了吗？我要拍最后一张合影！"
