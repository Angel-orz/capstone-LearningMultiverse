## 对话起点

课前复习 MCP 调试方法（stdio/SSE 正确，Inspector 需补充）。三月以积木类比引入 Skill 概念——每个 Skill 是一块积木，有四个面：Trigger、Instruction、Tools、Examples。

## 关键推理链条

```
Skill 四要素：Trigger → Instruction → Tools → Examples
  ↓
两条构建路径：翻译已知（vocab-review Skill） vs 结晶新知
  ↓
三层分工：AGENTS.md（你是谁）→ MCP（用什么做）→ Skill（怎么做）
  ↓
操作被调用范围控制 → 语句总结需要设定调用条件
  ↓
评估三维度：输出质量 / 工具轨迹 / Token 预算
```

## 核心概念

| 概念 | 掌握程度 | 备注 |
|------|---------|------|
| Skill 解剖结构（Trigger/Instruction/Tools/Examples） | 概念 ✓ / 应用 ✓ | 自主设计完整 Skill |
| 两条构建路径（翻译已知 vs 结晶新知） | 概念 ✓ / 应用 ✓ | Translating Known Workflows / Crystallizing New Capabilities |
| Skill vs MCP vs AGENTS.md 三层分工 | 概念 ✓ / 应用 ✓ | 学生自主判断"
| 评估三维度（输出质量/工具轨迹/Token 预算） | 概念 ✓ / 应用 ~ | 概念理解 |
| 工具轨迹分析方法 | 概念 ✓ / 应用 ~ | 调用顺序、参数、Token 消耗 |

## 学生表现

- 自主设计了完整的 vocab-review Skill（翻译已知路径）
- 正确判断诚实原则应放在 AGENTS.md 而非 Skill 中
- 识别出"语句总结"是最容易导致不必要 Token 消耗的工具——主动设定了调用条件
