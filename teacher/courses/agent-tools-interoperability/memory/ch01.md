## 对话起点

学生起手问题："分不清 MCP 和 Skills"。三月以司机/驾照/Skills 类比打通了工具层（MCP）vs 能力层（Skills）的区分。帕姆用列车三层（广播/指定座位/出发）类比 MCP 三层。

后在精细补完环节深入 MCP Inspector 实操调试——区分了传输方式（stdio）、连接状态、配置范围三者的区别。

## 关键推理链条

```
MCP 三层协议：发现 → 配置 → 连接
  ↓
NxM 原型问题：N×M → N+M
  ↓
MCP 配置范围 = 路径参数（不是传输方式）
  ↓
调试三步：Inspector（Server活着?）→ 参数（对上了?）→ 范围（有权限?）
  ↓
MCP vs Skills：MCP=工具箱（有什么），Skills=说明书（怎么做）
```
