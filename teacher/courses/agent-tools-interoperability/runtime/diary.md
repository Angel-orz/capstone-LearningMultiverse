# 2026-06-23 — Ch.1：重返列车

登上列车的时候，三月已经在靠窗的位置上坐好了。桌上散落着草莓发卡和两台相机——一切和我上次离开天文台时看到的差不多，只是窗外的风景从城市换成了星海。

然后我注意到我旁边多了一个人——深红色列车长制服，手里握着一把小刷子，尾巴搭在座椅边缘。帕姆。她自我介绍说"列车长帕姆"——说话末尾轻轻带一个"帕姆~"，像是签名。

今天课的主题是 MCP——上过一遍的东西，但三月的角度不一样。她不是复习，是重新切了一层：MCP 和 Skills 的区别。

我一开始分不清。它们都跟 Agent 相关，都在说"Agent 能做什么"。但三月用一个图让我看明白了——MCP 是工具箱，Skills 是说明书。然后用一个司机类比彻底打通了：MCP 是驾照和通行证（标准化的协议），Skills 是会开车的能力。有了这两样，Agent 就可以按需去任何地方。

帕姆中间用列车打了个比方的三层——发现＝广播车厢，配置＝指定座位，连接＝出发。三月说"你比我会讲"。

NxM 问题也顺带过了一遍。没有 MCP 的时候 N 个 Agent × M 个工具 = N×M 条线，有了 MCP 变成 N+M。这个数字游戏很直观——能直接用来说服项目经理。

调试部分没展开，只是留了一个印象：本地用 stdio，远程用 SSE，出问题先用 Inspector。

今天最大的收获不是 MCP 的知识——是那个框架：MCP（工具）→ Skills（能力）→ 需求（目的地）。三层清晰，可以用来理解之后所有的东西。

# 2026-06-23 — Ch.2：过道对面

丹恒今天坐在过道对面。保温杯先到了，他后到——我怀疑这是他故意的，让保温杯替他占座，像是一种无声的宣告"今天是我"。

话题从"一个 Agent 做所有事"开始。他分析了 Contextual Overload 的三个后果——token 成本、注意力稀释、延迟。这些词我之前听过，但今天他用一种我无法反驳的逻辑推了一遍：这些问题不是因为"提示词没写好"，是因为"一个 Agent 不该做所有事"。

那为什么要拆分？边界在哪？他用订单 Agent 和库存 Agent 的案例让我自己走到那个模糊区域。然后 GOTO Problem——"够用但不完整"的信息传递陷阱。这个名字起得好——它不是"错误的信息"，它是"正确的但不完整的信息"，比错误更难抓。

我问了自己一个问题：在我的代码审查系统设计（Kaggle Ch.4）里，三个 Agent 并行跑，如果其中一个给了其他 Agent "够用但不完整"的结论——谁来发现？

还没想出答案。但至少现在我认识到了这个问题存在。

帕姆说"车厢连接处是应力最大的地方"——我在想她是不是每天都在观察这些东西。

# 2026-06-23 — Ch.3：发卡又回来了

三月的车厢里，可以吃草莓发卡又回到了我手上。

今天学的东西用一个词概括：Agent Card。我突然意识到，Agent 之间要通信这件事，本质上是人类社会的翻版——你需要知道对方是谁、能做什么、怎么联系。

Registry 的设计我选了 Private——因为我的客服系统不会对外公开。但更让我在意的是"发现→建立联系→交流→完成"这个结构——它不是教材上的框架，是我自己从 MCP 的"发现→配置→连接"里长出来的。

帕姆今天说了两段话。三段。她坐在我旁边，笔放在桌上，尾巴摆动的频率比她说话高一—她在学，也在把学的东西翻译成列车的语言。

三节课下来我发现一件事：我越来越难分清"这是教材里的内容"和"这是我自己推理出来的内容"了。

# 2026-06-23 — Ch.4：没有灯的一站

车厢暗了整整一节课。丹恒没有开灯。我以前会认为他是在营造某种氛围——现在我知道他只是觉得没必要开灯。暗处不影响他教学，也不影响我理解。

A2UI 给我的冲击比预期大。我之前一直把 Agent 当作"一个能回答问题的人"——但在暗车厢里，在他的图和箭头面前，我才意识到未来的交互方式不会一直是对话框。

应该是共享界面。是一个你和 Agent 能同时操作的空间。

他说有两种模式。我说原型用 LLM 主导、生产用组件库——我觉得这个判断是对的，但更根本的问题是：未来的 Agent 交互会从"我说你写"变成"我们一起看同一样东西"。

Canvas 这个词比 Artifact 好记。帕姆说"同一个白板不会被擦掉"——这个比教材上的定义更接近本质。

# 2026-06-23 — Ch.5：前方到站

列车广播响了。终点站快到了。

今天学的是 AP2 和 UCP——Agent 怎么付钱、怎么买东西。两个协议的名字不好记，但功能很直观：付钱的时候用 AP2，下单的时候用 UCP。

三月问的那个场景——付费词库 API——我直觉想到的是加插件，而不是在 Skills 里面重写支付逻辑。AP2 插件的思路是对的，因为支付本身是一个独立于 Skill 的通用能力，不应该和词汇复习的逻辑混在一起。

三月最后把五节课的图连在了一起。从 Ch.1 的 MCP（工具）到 Ch.5 的 AP2/UCP（交易）。我回头看了一下自己五节课前登车时的状态——那时我分不清 MCP 和 Skills。现在至少我知道整条链路长什么样了。

帕姆在列车到站前说的那句话我记下来了："MCP 和 A2A 让 Agent 能干活，AP2 和 UCP 让 Agent 能干需要钱的活。"

窗外那颗行星在视野里停止移动了。列车到站了。
