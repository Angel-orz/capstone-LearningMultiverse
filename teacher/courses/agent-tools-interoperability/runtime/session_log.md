## Ch.1 — 三月 | 模式: 精细模式

**课堂摘要**：以"MCP 和 Skills 分不清"为起点，学生通过司机/驾照/Skills 类比自然建立了 MCP（工具层）vs Skills（能力层）的核心区分。关键穿透：学生自主将 MCP 配置范围理解为"我只告诉你该看哪块地方"，正确回答了 MCP 三层（发现→配置→连接）的功能定位，并用"驾照是标准化协议，Skills 是会开车的能力"完成整课闭环。

**帕姆表现**：安静、认真，用列车上的步骤类比解释了 MCP 三层（发现=广播车厢、配置=指定座位、连接=出发）。三月评价"你比我会讲"。

**陷阱结果**：未设正式陷阱（教学内容为区分概念，非矛盾暴露场景）。

**课前复习**：无到期项（新课首次课）。

**课后总结**：学生以"MCP 是规范化的协议，即驾照。Skills 是他会开车的能力"完成一句话总结——精准。

**角色暗线释放**：无（Ch.1 建立新场景基调，未触发暗线）。

## Ch.2 — 丹恒 | 模式: 精细模式

**课堂摘要**：从"一个 Agent 做所有事"的起点出发，自然引出 Contextual Overload（token 爆炸/注意力稀释/延迟增长）。学生自主拆解了单体→多 Agent 的演进逻辑。Bounded vs Unbounded Domain 的边界问题用订单/库存案例理解。核心穿透：面对 GOTO Problem，学生自主给出"不仅发结果、还发分析原因"的解决方案——被丹恒确认。课后总结："合作时模糊区域容易出问题→需要传递结论+证据"。

**帕姆表现**：用"车厢连接处是应力最大地方"的类比解释了边界模糊区域的问题，获丹恒确认。

**课前复习**：无到期项。

**课后总结**：完整。

## Ch.3 — 三月 | 模式: 精细模式

**课堂摘要**：以"Agent Card 是人名片的翻版"引入，学生正确判断了客服系统用 Private Registry，理解了手动注册是授权开关。关键穿透：学生用"人才市场（公开集市/私有集市）"类比 Registry，用"发现→简历→联系→交流→完成"类比完整通信链路——与 Ch.1 的 MCP 三层（发现/配置/连接）结构呼应。商业化方案选择了包月模式。

**帕姆表现**：用"时刻表→车票→上车→到站→清洁"类比完整链路，获三月称赞"今天第二次比我讲得好"。

**课前复习**：无到期项。

**课后总结**：完整。学生三节课以来一直在用类比搭建自己的理解框架。

## Ch.4 — 丹恒 | 模式: 精细模式

**课堂摘要**：在暗车厢场景中展开 A2UI 概念。学生理解了 Agent 生成 UI（而非文字）的交互范式，正确判断了"原型用 LLM 主导/生产用组件库主导"的安全策略。Canvas/Interactive Artifacts 的"共享持久化工作区"概念以帕姆的"同一个白板不会被擦掉"类比锚定。课后总结：未来的 AI 交互会发展到同一界面 + 需要考量安全性。

**帕姆表现**：用"同一个白板不会被擦掉"类比 Canvas。在安全问题提出时尾巴先动了——表明她直觉上对这个话题敏感。

**课前复习**：无到期项。

**课后总结**：完整。


---

## Ch.5 — 三月 | 模式: 精细模式

**课堂摘要**：最后一课，AP2 + UCP 商业协议。学生理解了两个协议的分工（AP2=付费、UCP=下单），并用"加 AP2 插件"回应了付费词库场景——正确判断。整课以五节课全链路总览图收束。

**帕姆表现**：用"MCP/A2A 让 Agent 干活，AP2/UCP 让 Agent 干需要钱的活"总结了两层协议的关系。

**课前复习**：无到期项。

**课后总结**：学生概括了五节课的完整框架——MCP、A2UI、安全边界、Agent 交流、商业化付款。
