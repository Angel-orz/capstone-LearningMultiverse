# Ch.1 — 重返列车

- **老师**：三月 🎀
- **情感阶段**：初期 — 信号
- **教材范围**：p6-16（MCP 回顾与深化）

## 场景

你登上列车的时候，三月已经在了。她占了靠窗的那一排座位——桌上散落着几颗草莓发卡、两台相机（一台胶片的，一台数码的）、一叠写了半截的便签。她看到你上来，一只手上还握着笔，另一只手已经举起来准备打招呼了。

"诶！你来啦！"

然后她看到你身后——一个穿着深红色列车长制服的身影跟了进来。帕姆走到你旁边的座位坐下，把你的座位让给你——她不放笔在那占座，因为这个车厢是她的，她知道你坐哪。

三月眨了两下眼——然后笑了。"好！今天人齐了！"

她把白板从车厢壁上翻下来——上面还留着上次的 MCP 结构图。她没擦。

"MCP——我们上次讲过的。但今天不是复习，是从一个新的角度看它。你上次用 MCP 的时候——你觉得最不顺手的地方是什么？"

她看了看你，又看了看帕姆。"你俩都可以说。"

## 故事节点

1. **开场**：重返列车——三月和她的散落装备、帕姆跟在身后入场
2. **MCP 三层的完整协议栈**：Host-Client-Server
3. **NxM 问题**：为什么需要 MCP——从"每个工具写一个连接器"到"一个协议走天下"
4. **调试实操**：Inspector / stdio / SSE 的适用场景
5. **最佳实践**：Vibe Coder Toolkit——内部 API 网关、访问范围控制

## 真实课堂记录

**2026-06-23 | 三月 🎀 | 精细模式 | Ch.1**

学生起点：分不清 MCP 和 Skills。全程用"司机/驾照/Skills"类比自主打通了工具层（MCP）vs 能力层（Skills）的区分。NxM 问题以图示理解，配置范围以路径参数说明理解。帕姆用列车三层（广播/指定座位/出发）类比 MCP 三层，获三月称赞。课后总结完美闭环。

## 过渡

三月没擦白板。她对帕姆说"明天丹恒来的时候你别告诉他这块板我没擦——让他自己先猜猜我们在说什么。"帕姆的尾巴轻轻摆了一下——她答应了，但她也知道丹恒根本不会问。
