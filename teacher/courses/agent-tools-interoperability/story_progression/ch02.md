# Ch.2 — 过道对面

- **老师**：丹恒 🗡️
- **情感阶段**：初期 — 信号
- **教材范围**：p17-23（A2A 架构演进、边界、GOTO）

## 场景

今天你上车时，三月的座位空着——但过道对面的座位上有一个保温杯。丹恒还没到，但保温杯已经到了——说明他来过，而且还会回来。

帕姆已经坐在你旁边的位置了——手里握着她的小刷子，尾巴尖安静地搭在座椅边缘。她扫了一眼桌面，确认今天没有需要清理的角落。

丹恒从车厢连接处走过来，手上端着一个新的保温杯（刚才那个是凉的——他先放过来占座，去接了热的）。他坐下时看了一眼帕姆——目光在她尾巴上停了半拍——不是在好奇，是在确认她已经准备好听课了。帕姆的尾巴轻轻摆了一下。

他在白板上画了一个很简单的图——一个方块分成两半。

"上一课三月带你们回顾了 MCP——工具怎么连接。今天的问题是：工具连接上了，然后呢？Agent 自己需要沟通。沟通需要结构——没有结构的沟通是噪音。"

## 故事节点

1. **从 MCP 到 A2A**：工具连好了，Agent 之间怎么说话
2. **架构演进**：一体的 Agent → 内部专业化（Internal Specialization）→ 外部专业化（External Specialization的）
3. **Contextual Overload**：把系统指令、工具定义、对话历史全部塞进一个提示词的后果
4. **Bounded vs Unbounded Domains**：Agent 知道自己什么能做什么不能做
5. **GOTO Problem**：Good-Enough-But-Not-All 的传递陷阱

## 真实课堂记录

**2026-06-23 | 丹恒 🗡️ | 精细模式 | Ch.2**

从 Contextual Overload 出发，学生自主拆解了单体→多 Agent 演进逻辑。边界案例（订单+库存）引出 GOTO Problem，学生主动给出"传证据不传结论"的解决方案。帕姆用"车厢连接处应力最大"类比边界模糊区域。

## 过渡

丹恒没有擦黑板。他把保温杯盖上，站起来时看了帕姆一眼——那个看法的意思不是"今天你表现得不错"——是他接受了她用列车连接处类比车厢连接处的这件事。帕姆的尾巴摆了一下，没有说"谢谢帕姆~"——她不需要说。
