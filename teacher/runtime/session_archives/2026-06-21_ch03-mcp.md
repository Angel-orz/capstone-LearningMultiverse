### 教学内容流程

1. **引入** — 三月用电饭煲 MCP 翻车现场开场，连接失败
2. **NxM 原型问题** — N 个模型 × M 个工具需要 N×M 个集成 → MCP 降到 N+M
3. **MCP 三步** — 发现（公开注册中心/官方/内部）→ 配置（参数/权限/认证）→ 连接（握手验证）
4. **JSON-RPC 2.0 协议** — stdio（本地进程） vs SSE（远程流式）
5. **学生追问①** — Context Engineering 六种上下文 vs Harness 七组件的相似与区分
   - 解答：CE 是输入层图纸，Harness 是执行层基础设施，重合不是 bug 是特性
6. **学生追问②** — 不理解 MCP 和传统 API 调用的区别及 MCP 配置
   - 重新解释：插线板类比、数据库例子分解、配置三问（接什么/什么权限/怎么认证）
   - 三月类比翻车：电饭煲插进插线板台灯亮了 → 学生指出逻辑断层 → 当面纠正并贴便签记录
7. **MCP 调试** — MCP Inspector（查工具列表/手动测试/看原始包）
8. **Vibe Coder Toolkit** — 最佳实践规则（Do/Don't）
9. **A2A 引入** — Agent-to-Agent 协议概念
   - MCP = 高速公路 + 收费站，A2A = 路上两辆车交流
   - Agent Card、Discovery、Task 生命周期管理
10. **学生总结** — "MCP是一种规格，A2A是一种工作形式"

### 关键学生表现

- 主动追问 MCP vs Context Engineering vs Harness 的结构关系
- 指出三月类比的逻辑断层（电饭煲插插线板台灯不该亮）
- 用"高速公路/收费站/两辆车"连贯类比自主建构 MCP 和 A2A 的区别
- 课后总结用八个字精准区分两层
