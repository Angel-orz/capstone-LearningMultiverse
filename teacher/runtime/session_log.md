# session_log.md — 课堂日志

> **文件压缩规则**（嵌入头部）：
> 当本文件超过 500 行时，将前 400 行追加到 `session_archive.md`，本文件保留最近 10 课的日志。
> 压缩时在 archive 文件追加分隔符 `--- {日期} 归档 ---`。

## 日志记录格式

每课课后写课堂摘要，包含：
- 课程ID + 教学模式
- 学生今日表现（亮点 + 卡点）
- 陷阱结果（踩了/没踩/部分踩 + 下节课密度建议）
- 🚨 最后一英里规则执行情况
- 课前复习表现（如有到期项）
- 课后总结记录（学生的自述 + 教师反馈）
- 角色暗线释放记录（如有）

日志长度：
- 粗略模式：100-150 字
- 精细模式：200-300 字

**课前复习**：无到期项。

**角色暗线释放**：丹恒展示了自己的手写笔记本——半年前关于Memory/Knowledge分类的思考笔记，与学生课上提出的分类标准一致。

**下节课建议**：三月准备用Day 2的MCP实操演示。"让学生先看到工具实际跑起来"比定义先行更适合学生当前的理解节奏。

## Ch.3 — 三月 | 课程: kaggle-agent | 模式: 精细模式

**课堂摘要**：以三月翻车的电饭煲MCP实操引入，学生自然抓住了MCP作为工具接口标准化的核心价值。关键推理链条：N×M集成问题 → MCP统一接口 → 发现/配置/连接三步 → JSON-RPC 2.0协议（stdio/SSE） → MCP Inspector调试。学生将MCP类比为"高速公路和收费站"，A2A类比为"路上两辆车交流"——并主动追问MCP与Context Engineering六种上下文/Harness七组件的关系。三月翻车了一个类比（电饭煲插插线板台灯亮了），当面纠正并记录。学生课后总结："MCP是一种规格，A2A是一种工作形式"——精准区分了两层的关系。

**课前复习**：4个到期项——学生正确回答了Vibe Coding光谱（架构）、Factory Model vs Harness区分（规划vs实现）、Harness 7组件（准确回忆了8个，含observability）、Context Engineering跳过（三月标记为"今天会自然串起"）。

**陷阱结果**：未设正式陷阱（学生主动提问MCP vs Context Engineering vs Harness的关系，提供了比预设陷阱更精准的教学切入点）。

**🚨 最后一英里规则**：不适用（讲解为主模式）。

**角色暗线释放**：三月自嘲翻车类比后贴了一张"类比翻车×1"便签在显示器边缘——这是她第一次在学生面前主动暴露自己的教学失误并亲手记录下来，比平时切换节奏时多了一拍停顿。

**下节课建议**：学生已建立MCP→A2A的过渡概念框架，Ch.4可由丹恒从Agent Card结构展开，配合Task生命周期的具体状态转换。学生类比能力强，建议多用"收费站→高速公路→行驶证→两车交流"的连贯类比链。
