# nuwa-integration.md — New Teacher Creation Bridge

> This file is loaded when "the teacher for the current course does not exist."
> Workflow: course_catalog specifies teacher → teacher/skills/{name}.skill.md does not exist
>   → check capabilities.md → also does not exist
>   → execute this bridge workflow to create a new teacher

---

## Trigger Conditions

When `teacher/skills/{teacher_name}.skill.md` does not exist, and the teacher is also not in `teacher/skills/capabilities.md`:

```
1. Confirm teacher_name (from course_catalog.md or learner_profile.md)
2. Confirm teaching subject and style (from course description in course_catalog.md)
3. Confirm whether to use real-person distillation or fictional creation
```

## Calling nuwa-skill

nuwa is installed in the `.claude/skills/nuwa/` directory.

**Calling Method** (choose one suited to the scenario):

### A) Distilling a Real Person

When transforming a real expert (e.g., a well-known figure in a subject area) into a teacher:

```
Skill("huashu-nuwa", args="Distill {person name} as a {subject} teacher for LearningMultiverse.
Output must be in teacher/skills/{name}.skill.md format.")
```

### B) Fictional Creation of a New Teacher

When creating a brand-new fictional teacher for a course:

```
Skill("huashu-nuwa", args="Create a fictional teacher for the LearningMultiverse course {course name},
named {name}, teaching style {style description}, subject {subject}.
Output must be in teacher/skills/{name}.skill.md format.")
```

## Output Conversion

The SKILL.md format generated by nuwa-skill may differ slightly from the `teacher/skills/` format.
After creation, **must** adjust the following fields:

1. **frontmatter**: Ensure it includes `type: teacher`, `source: nuwa-distilled`, `capabilities:`
2. **config**: Add `partner:` (rotation partner, optional), `chapter_pattern:` (lesson regularity)
3. **Save path**: Must be `teacher/skills/{name}.skill.md`

## Registration

After creation, add a row to the registration table in `teacher/skills/capabilities.md`:

```
| {name} | {display name} | fictional | nuwa-distilled | {tags} |
```

## First Use

After the new teacher is registered:
1. This session directly loads the newly created skill file
2. Notify the user "Teacher {name} is ready"
3. Subsequent sessions automatically follow the normal loading path
